Spreadsheet Add Column Bug Analysis

The Problem

The add column button in SingleSpreadsheet.tsx has a critical bug. Let's examine the problematic code: 
onClick={() => {
  // add an empty cell to each row
  const spreadsheetRows = [...spreadsheet.rows];
  for (let i = 0; i < spreadsheet.rows.length; i++) {
    spreadsheet.rows[i].push({ value: "" });  // ❌ BUG: Modifying original array!
  }
  setSpreadsheet({
    ...spreadsheet,
    rows: spreadsheetRows,
  });
}}

Why It Fails

  1. Shallow Copy Issue: [...spreadsheet.rows] creates a shallow copy of the rows array, but each row is still a reference to the original row array.
  2. Direct Mutation: The code modifies spreadsheet.rows[i] directly, which mutates the original state.
  3. React State Update: React may not detect the change because the array references haven't changed (only their contents have).

Interactive Demo

The Solution

onClick={() => {
  // add an empty cell to each row
  const spreadsheetRows = spreadsheet.rows.map(row => [...row, { value: "" }]);
  setSpreadsheet({
    ...spreadsheet,
    rows: spreadsheetRows,
  });
}}

Alternative Solutions

// Option 2: Deep clone approach
onClick={() => {
  const spreadsheetRows = spreadsheet.rows.map(row => [...row]);
  for (let i = 0; i < spreadsheetRows.length; i++) {
    spreadsheetRows[i].push({ value: "" });
  }
  setSpreadsheet({
    ...spreadsheet,
    rows: spreadsheetRows,
  });
}}
// Option 3: Using structuredClone (modern browsers)
onClick={() => {
  const spreadsheetRows = structuredClone(spreadsheet.rows);
  for (let i = 0; i < spreadsheetRows.length; i++) {
    spreadsheetRows[i].push({ value: "" });
  }
  setSpreadsheet({
    ...spreadsheet,
    rows: spreadsheetRows,
  });
}}

Expected Console Errors

When clicking the add column button with the current bug, you might see:
  • No immediate errors, but the spreadsheet won't update visually
  • React development mode warnings about state mutations
  • Potential "Cannot update a component while rendering" warnings
  • The onChange handler logs showing data changes, but UI not reflecting them
